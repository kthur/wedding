# 💍 결혼 준비 앱 (Wedding Planner) — 최종 구현 계획

## 개요

커플이 함께 결혼 준비를 관리할 수 있는 크로스플랫폼 모바일 앱입니다.  
남자/여자 각각이 로그인하여 파트너와 연동되면 모든 정보를 실시간으로 공유합니다.  
최종 데이터는 Excel / PDF로 내보낼 수 있습니다.

---

## UX 설계 원칙

> 결혼 준비는 **스트레스가 높은 프로젝트**입니다.  
> 앱은 **"지금 뭘 해야 하지?"** 라는 질문에 즉시 답해줘야 합니다.

1. **홈에서 모든 핵심 정보** — D-Day, 진행률, 다음 일정을 한눈에
2. **두 가지 뷰 모드** — 카테고리별 뷰 ↔ 시기별 체크리스트 뷰 토글
3. **빠른 상태 변경** — 카드에서 탭 한 번으로 상태 업데이트
4. **파트너 동기화 표시** — 상대방이 언제 무엇을 수정했는지 표시
5. **업체 연락처 통합** — 모든 카테고리에서 전화/문자 바로 연결
6. **사진 갤러리** — 각 카테고리별 추억 사진 모아보기

## 화면 구성 — 5개 탭

```
 홈   준비   일정   예산   더보기
  🏠    📋    📅    💰    ···
```

### Tab 1 — 홈 (Home)
- **D-Day 카운트다운** (크게, 중앙 상단)
- **전체 진행률** — 도넛 차트 (완료/진행중/미시작 비율)
- **파트너 업데이트** — "OO님이 [결혼식장]을 수정했습니다 · 2시간 전"
- **다가오는 일정** — 최대 3개, 날짜순 카드
- **커플 메모** — 파트너에게 남기는 한 줄 메시지

## Firestore 데이터 구조

```
couples/{coupleId}/
  ├── info/
  │     ├── maleUid / femaleUid
  │     ├── weddingDate
  │     └── budgetGoal
  ├── categories/{categoryId}/        ← 24개 카테고리
  │     ├── status                    (none | inProgress | done)
  │     ├── estimatedCost / actualCost
  │     ├── notes
  │     ├── vendorName / vendorPhone
  │     ├── schedules: [{id, date, title, reminderDays}]
  │     ├── photos: [{url, caption, uploadedBy, uploadedAt}]
  │     └── updatedBy / updatedAt
  ├── timeline_checklist/{itemId}/    ← 시기별 체크리스트
  │     ├── phase                     (D-6m | D-5m | D-3m | D-1m | D-2w)
  │     ├── title / isDone
  │     ├── linkedCategoryId          (카테고리 연동)
  │     └── createdBy
  ├── checklist/{itemId}/             ← 자유 체크리스트
  │     ├── title / isDone / createdBy
  ├── memos/{memoId}/
  │     ├── text / sentBy / createdAt
  └── guests/{guestId}/
        ├── name / phone / side       (groom | bride)
        └── mealConfirmed / attended

users/{uid}/
  ├── name / gender / coupleId / inviteCode

storage/couples/{coupleId}/{categoryId}/{photoId}.jpg
```

## GitHub Actions CI/CD (바이너리 생성)

GitHub Actions를 통해 코드가 푸시되거나 릴리즈 태그가 생성될 때 자동으로 플랫폼별 바이너리(Android APK, Windows EXE 등)를 빌드하여 생성하는 워크플로우를 구축합니다.

### 워크플로우 명세 (`.github/workflows/build.yml`)
- **트리거**: `main` 브랜치로의 `push` 또는 `v*` 형태의 릴리즈 태그 푸시 시 작동
- **빌드 대상**:
  1. **Android**: `apk` (release용) 및 `appbundle` (AAB) 생성
  2. **Windows**: Windows용 `executable` (.exe) 빌드
- **자동 배포**:
  - `v*` 태그가 생성되면 자동으로 GitHub Release를 생성하고 빌드된 APK 및 Windows 패키지를 첨부합니다.
